# 标注协作看板 PRD 示例

## 1. 背景与目标

本原型用于说明 Axhub Make 的标注能力如何把可运行原型、节点说明、状态控制和补充文档放在同一个评审入口里。它不是一份静态说明页，而是一个可以切换页面、点击标注、查看目录和阅读文档的演示项目。

目标是让评审者理解：原型即 PRD 负责承载界面和交互，标注负责补充边界、原因和决策，目录负责组织页面、文档和外部来源。这样产品、设计、研发和后续 Agent 可以围绕同一份上下文工作，减少原型、PRD、截图和聊天记录之间的重复同步。

## 2. 功能范围

- 原型即 PRD：解释为什么可运行页面可以成为需求主载体，并用对比表说明它相对传统原型 + PRD 的效率优势。
- 内容标注：演示任意元素标注、Markdown 正文、多颜色分类、颜色筛选和同一节点多条标注。
- 状态标注：演示结果状态、列表状态和指标状态，评审者可以在标注面板中切换不同业务状态。
- 原型目录：把页面、PRD 文档和外部设计来源放进同一棵目录树，减少评审时找资料的成本。
- 开启标注：说明可以通过 AI 或手动开启标注，推荐 AI 方式。
- 编辑标注：说明 AI 编辑和手动编辑两种方法。
- Agent 读取：说明开发 Agent 读取源码、标注内容、文档内容和截图，用于后续开发。

## 3. 不在本次范围

本演示不覆盖真实后端接口、用户权限系统、多人实时协作冲突处理和正式发布审批流。它聚焦标注运行时本身，以及目录如何承载围绕原型的产品说明。

## 4. Markdown 层级案例

### 背景

用于确认 Markdown 链接目录在文章正文中是否按普通链接展示。

### 范围

目录项需要支持一级和二级缩进，方便长文档快速定位章节。

#### 页面范围

页面章节用于描述入口、页面切换和节点位置。

#### 状态范围

状态章节用于描述默认状态、可切换分支和边界说明。

### 验收

点击或查看目录时，链接文案、层级缩进和标题结构都应保持可读。

### 4.2 多级列表

1. 一级事项：评审入口
    1. 二级事项：页面切换
        - 三级事项：保留当前标注上下文
        - 三级事项：确认 hash route 可定位
    2. 二级事项：目录文档
        - 三级事项：展示 PRD 和验收清单
2. 一级事项：标注阅读
    - 二级事项：按颜色筛选
        - 三级事项：只看风险或待确认项
    - 二级事项：按节点查看
        - 三级事项：聚焦当前激活节点
