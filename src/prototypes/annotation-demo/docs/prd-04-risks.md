# 风险

## 1. 已知风险

- 信息过载：如果每个节点都堆叠多条说明，评审者会失去重点。需要通过颜色筛选、标题和目录分层控制阅读顺序。
- 状态遗漏：如果只展示成功路径，研发容易忽略失败、空数据和指标异常。需要用状态标注补全关键分支。
- 文档漂移：PRD 文档如果长期不更新，会重新变成与原型脱节的资料。重要变更应优先回到页面和节点标注。
- 链接膨胀：外部链接过多会稀释目录价值，本演示只保留必要的设计来源。

## 2. 控制策略

内容标注要围绕关键节点，不追求覆盖所有文字。多颜色标注只用于功能性分类，不改变整体 Swiss 视觉主题。目录文档按照 PRD 00-05 组织，确保每篇文档都有明确章节和用途。

## 3. 待确认事项

后续真实项目需要确认是否开放多人共同编辑、是否需要标注版本历史、是否需要导出为独立文档，以及状态标注是否要接入真实业务数据。
