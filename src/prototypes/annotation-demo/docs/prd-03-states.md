# 状态

## 1. 状态标注范围

状态标注用于把业务条件和页面表现绑定在同一个节点上。本演示覆盖三类状态：结果状态、列表状态和指标状态。

## 2. 状态清单

- 结果状态：成功时展示保存完成和下一步协作提示；失败时展示网络异常和重试提示。
- 列表状态：空列表时说明下一步入口和业务原因；有内容时展示最近更新的标注任务说明。
- 指标状态：偏低时提示排查方向；正常时保留业务含义；偏高时强调峰值原因和潜在风险。

| 状态区域 | 默认值 | 可切换分支 | 验收关注 |
| ---- | ---- | ---- | ---- |
| 结果状态 | 成功 | 成功 / 失败 | 文案、图标和下一步操作需要随状态同步变化 |
| 列表状态 | 空列表 | 空列表 / 有内容 | 空态要说明业务原因，有内容时要展示最近更新项 |
| 指标状态 | 正常 | 偏低 / 正常 / 偏高 | 指标阈值、趋势文案和风险提示不能互相冲突 |

## 3. 默认状态

结果状态默认成功，列表状态默认空列表，指标状态默认正常。默认值用于保证首次打开页面时有稳定演示效果，评审时再通过标注面板切换不同分支。

## 4. 验收要求

每个状态项都应有清晰名称、可读文案和明确业务含义。状态切换后，页面内容应同步变化，标注说明不应与当前状态冲突。
