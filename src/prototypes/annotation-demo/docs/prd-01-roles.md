# 角色

## 1. 使用者

- 产品负责人：负责确定页面目标、业务边界、验收标准和标注内容是否表达清楚。
- 设计师：负责确认页面结构、视觉规则、交互状态和设计来源是否统一。
- 研发：负责根据可运行原型和节点标注理解页面行为、状态条件和实现优先级。
- 评审者：负责在目录中按页面和文档阅读上下文，并通过标注理解问题、风险和决策。
- Agent：负责基于页面结构、需求资料和已有规则生成第一版标注、目录和补充文档。

## 2. 协作职责

产品负责人先定义原型要表达的功能和评审重点；设计师补充视觉约束和交互细节；研发读取页面与标注来判断实现方式；评审者通过目录快速进入相关页面或文档；Agent 则负责把重复性说明批量生成出来，再交给人工校正。

## 3. 权限与边界

目录中的页面和文档默认用于评审阅读。标注编辑、状态配置和生成结果确认应由项目维护者完成；外部链接只保留必要来源，避免把目录变成资料堆。
