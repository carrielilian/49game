# 标记付款抽屉

## 字段

| 字段 | 规则 |
|------|------|
| 游戏ID、游戏名称 | 只读。**初始**：打开时展示当前付款记录对应游戏 |
| 待付款金额 | 只读；￥ + 千分位。**初始**：打开时展示该笔记录的待付款金额 |
| 实际付款金额 * | 可编辑；￥；必填；≥ 0；精确至小数点后两位。**初始**：见下方「金额初始计算」 |
| 实际付款美金 | 可编辑；$；选填；≥ 0。**初始**：见下方「金额初始计算」 |
| 分成付款公司 | 只读；取游戏收入【付款设置】。**初始**：展示已配置的分成付款公司（申请付款前须已在【付款设置】维护） |
| 收款信息 * | 多行；必填。**初始**：该笔记录已保存过则显示已保存内容；否则默认填入归属厂商银行五字段 |
| 备注 | 选填。**初始**：该笔记录已保存过则显示已保存内容；否则为空 |

## 金额初始计算

**每次打开**【标记付款】都会按最新数据重算并写入输入框；写入后仍可手动修改。

### 参与变量

| 变量 | 来源 |
|------|------|
| 待付款金额 | 本笔付款记录 |
| 预付分成款 | 游戏收入【付款设置】 |
| 剩余未抵扣分成款 | 按游戏当前预付数据计算 |
| 合同支付币种 | 游戏合同支付币种（预付分成款币种） |
| 付款币种 | 游戏收入【付款设置】 |
| 汇率 | 申请时间的**上一个月**月末汇率 |

### 中间量

| 名称 | 计算方式 |
|------|----------|
| net | 待付款金额 − 剩余未抵扣分成款；若 ≤ 0 则按 0 |
| usdNet | 待付款金额 ÷ 汇率 − 剩余未抵扣分成款；若 ≤ 0 则按 0 |

### 五种情况

| 情况 | 条件 | 实际付款金额（￥） | 实际付款美金（$） |
|------|------|-------------------|------------------|
| ① | 预付分成款 ≤ 0，付款币种 = 人民币 | = 待付款金额 | 空 |
| ② | 预付分成款 ≤ 0，付款币种 = 美金 | = 待付款金额 | = 待付款金额 ÷ 汇率 |
| ③ | 预付分成款 > 0，合同支付币种 = 人民币，付款币种 = 人民币 | = net | 空 |
| ④ | 预付分成款 > 0，合同支付币种 = 人民币，付款币种 = 美金 | = net | = net ÷ 汇率 |
| ⑤ | 预付分成款 > 0，合同支付币种 = 美金 | = usdNet × 汇率 | = usdNet |

全额抵扣后 net / usdNet 可能为 0，此时实际付款金额显示 **0.00**。

## 按钮

| 按钮 | 行为 |
|------|------|
| 取消 | 关闭，不保存 |
| 标记已付款 | 校验通过后保存并改状态为已付款；关闭抽屉；绿色「提交成功」 |

校验失败：字段下红字 + 红 Toast「请完善所有信息」。
