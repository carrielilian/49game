# 结算函抽屉

## 做什么

1175px 宽抽屉，展示本次付款对应的「合作分成结算确认函」；只读预览，可下载中文 / 英文 PDF。

## 顶部

| 项 | 说明 |
|----|------|
| 标题 | {厂商名称} 与 四三九九网络股份有限公司 合作分成结算确认函 |
| 下载 | 下拉选择中文或英文，生成可打开的 PDF 模拟文件 |

## 数据来源

| 情况 | 表现 |
|------|------|
| 未付款 | 按本笔付款关联的已结算明细实时生成 |
| 已付款 | 读取标记已付款时冻结的快照，不再随外部数据变化 |

明细来自申请付款时纳入的结算记录；同一游戏同一渠道且月份连续的多行会合并为一行。

## 收入区

| 列 | 说明 |
|----|------|
| 合作产品名称 | 游戏名称 + 渠道 |
| 结算时间 | 如 2025.06 或 2025.05-2025.06 |
| 游戏产品收入① | 待结算金额；￥ + 千分位 |
| 结算公式 | 公式中「待结算金额」显示为 ① |
| 结算金额 | 已结算的结算收入；￥ + 千分位 |

**合计②**：收入区结算金额之和。

## 退款区

无退款明细时，**整段退款区不显示**（含表头、合计④、总计②-④三行）。

有退款明细时：

| 列 | 说明 |
|----|------|
| 合作产品名称 | 游戏名称 + 渠道 |
| 退款时间 | 月份格式同收入区 |
| 退款金额③ | 待结算退款；￥ + 千分位 |
| 结算公式 | 公式中「待结算金额」显示为 ③ |
| 结算退款 | 已结算的结算退款；￥ + 千分位 |

**合计④**：退款区结算退款之和。

**总计②-④**：② − ④。

## 汇总区

| 行 | 说明 |
|----|------|
| 汇率 | **仅付款币种为美金时**显示；取申请时间的**上一个月**月末汇率，四位小数 |
| 结算/抵扣金额 | **仅剩余未抵扣分成款 > 0 时**显示；见下方⑤规则 |
| 剩余未抵扣预付分成款 | 同上条件；= 剩余未抵扣 − ⑤ |
| 实际付款金额 | 始终显示；见下方「实际付款金额」 |
| 支付金额（大写） | 与实际付款金额对应的中文大写 |

### ⑤ 结算/抵扣金额

设 remaining = 游戏「剩余未抵扣分成款」（与游戏收入列表同源）。

| 条件 | 结算/抵扣金额 |
|------|----------------|
| remaining − (②−④) > 0 | = ② − ④ |
| 否则 | = remaining |

函内剩余未抵扣预付分成款 = remaining − 结算/抵扣金额。

### 实际付款金额（游戏付款）

与【标记付款】初始填充口径一致（见「标记付款抽屉」批注五分支）；**不是** ②−④−⑤。

| 付款币种 | 展示 |
|----------|------|
| 人民币 | ￥ + 实际付款金额 |
| 美金 | $ + 实际付款美金 |
