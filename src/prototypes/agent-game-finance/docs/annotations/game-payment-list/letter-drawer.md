# 结算函抽屉

## 做什么

1175px 宽抽屉，展示本次付款对应的「合作分成结算确认函」；只读预览，可下载中文 / 英文 PDF。

## 顶部

| 项 | 说明 |
|----|------|
| 标题 | {厂商名称} 与 四三九九网络股份有限公司 合作分成结算确认函 |
| 下载 | 下拉选择中文或英文，生成可打开的 PDF 模拟文件 |

## 数据来源

**申请付款成功时**，系统即按当时信息生成并保存本函全部内容（收入/退款明细、合计、汇率、抵扣行、实付等）。之后无论付款状态是「未付款」还是「已付款」，打开结算函**均读取该快照**，不再随汇率、预付、付款设置等外部数据变化。

明细来自申请付款时纳入的结算记录；同一游戏同一渠道且月份连续的多行会合并为一行。

**标记已付款**时，结算函内容**全部保持不变**（含「实际付款金额」）；标记付款表单中的实付仅写入付款记录，供列表/详细信息展示。

## 收入区

| 列 | 说明 |
|----|------|
| 合作产品名称 | 游戏名称 + 渠道 |
| 结算时间 | 如 2025.06 或 2025.05-2025.06 |
| 游戏产品收入① | 待结算金额；￥ + 千分位 |
| 结算公式 | 公式中「待结算金额」显示为 ① |
| 结算金额 | 已结算的结算收入；￥ + 千分位 |

**合计②**：收入区结算金额之和。

## 退款区

无退款明细时，**整段退款区不显示**（含表头、合计④、总计②-④三行）。

有退款明细时：

| 列 | 说明 |
|----|------|
| 合作产品名称 | 游戏名称 + 渠道 |
| 退款时间 | 月份格式同收入区 |
| 退款金额③ | 待结算退款；￥ + 千分位 |
| 结算公式 | 公式中「待结算金额」显示为 ③ |
| 结算退款 | 已结算的结算退款；￥ + 千分位 |

**合计④**：退款区结算退款之和。

**总计②-④**：② − ④。

## 汇总区

| 行 | 说明 |
|----|------|
| 汇率 | 见下方「汇率行」 |
| 结算/抵扣金额 | **仅剩余未抵扣分成款 > 0 时**显示；见下方⑤规则；币种同**支付币种** |
| 剩余未抵扣预付分成款 | 同上条件；币种同**支付币种** |
| 实际付款金额 | 始终显示；见下方「实际付款金额」 |
| 支付金额（大写） | 与实际付款金额对应的中文大写 |

### 汇率行

满足**任一**条件时显示（汇率取申请时间上一个月月末，四位小数）：

1. **付款币种**为美金（【付款设置】中配置）
2. **支付币种**为美金（申请付款快照），且当时「剩余未抵扣分成款」大于 0

### ⑤ 结算/抵扣金额

**前提**：申请付款时，游戏「剩余未抵扣分成款」大于 0，才显示本行及下一行。

**支付币种**：【申请付款】那一刻的快照值。

**剩余未抵扣分成款**：【申请付款】那一刻的快照值（与当时游戏收入列表一致）。

**总计②-④**：本函保存的合计（申请付款时确定）。

按**支付币种**分两种算法：

**支付币种 = 人民币**

| 条件 | 结算/抵扣金额 | 剩余未抵扣预付分成款 |
|------|--------------|---------------------|
| 总计②-④ − 剩余未抵扣分成款 ≥ 0 | = 剩余未抵扣分成款 | 0 |
| 否则 | = 总计②-④ | 剩余未抵扣分成款 − 总计②-④ |

**支付币种 = 美金**

先将总计②-④按本函汇率折算为美元，再与剩余未抵扣分成款（美元）比较：

| 条件 | 结算/抵扣金额 | 剩余未抵扣预付分成款 |
|------|--------------|---------------------|
| 折算美元 − 剩余未抵扣分成款 ≥ 0 | = 剩余未抵扣分成款 | 0 |
| 否则 | = 折算美元 | 剩余未抵扣分成款 − 折算美元 |

⑤ 与剩余行按**支付币种**展示（人民币 ￥、美金 $）。**汇率行**与**实际付款金额**按**付款币种**（【付款设置】中的付款币种）。

### 实际付款金额（游戏付款）

与【标记付款】初始填充口径一致（见「标记付款抽屉」批注三种情况）；**不是** 总计②-④ 减⑤。

| 付款币种 | 展示 |
|----------|------|
| 人民币 | ￥ + 实际付款金额 |
| 美金 | $ + 实际付款美金 |

始终显示申请付款时按规则算出的金额；**标记已付款后不改变**（与标记付款表单填写值无关，表单值仅在付款记录/列表中展示）。
