# 申请付款状态列

## 做什么

标识该条结算记录是否已纳入「申请付款」，表头可按此状态筛选列表。

## 状态含义

| 状态 | 什么时候显示 |
|------|----------------|
| **未申请** | 该条记录**尚未**被任何一次成功的「申请付款」纳入。包括：刚导入/拉取/结算完成的新记录，以及尚未在游戏收入中发起申请付款的记录 |
| **已申请** | 该条记录已在**游戏收入**中「申请付款」成功，被纳入当次付款申请 |

## 与账户余额的关系

只有状态为「未申请」且已完成结算的记录，才会计入对应游戏的**账户余额**。变为「已申请」后，不再计入余额。

## 可选项（筛选）

未申请、已申请。

## 不同情况怎么展示

| 情况 | 列表表现 |
|------|----------|
| 未选筛选 | 显示全部 |
| 选中某一状态 | 只显示匹配行 |
| 单元格 | 彩色状态标签（未申请为橙色，已申请为绿色） |

## 示例

- 外部收入刚「确认导入」的记录 → **未申请**
- 已在游戏收入中对应该游戏点击「申请付款」并成功 → 该游戏相关记录 → **已申请**
