# 操作记录抽屉

## 展示内容

顶部只读：游戏编号 / 游戏名称。

下方表格三列：操作人、操作时间、操作。

## 哪些变更会写入操作记录

| 来源 | 何时写入 | 记录类型 |
|------|----------|----------|
| 添加游戏 | 新建游戏保存成功 | 添加游戏 |
| 编辑游戏 | 「运营状态」与保存前不同 | 运营状态 |
| 合同管理 | 「合作状态」与保存前不同 | 合作状态 |
| 合同管理 | 见下表「合同变更字段」任一发生变化 | 合同变更 |

### 合同变更：会记哪些字段

保存合同时，仅以下字段**前后值不同**时，合并写入一条「合同变更」记录；「操作」列按字段分行展示：

| 字段 | 说明 |
|------|------|
| 合同金额 | 金额带币种符号与千分位 |
| 已付游戏代理金 | 仅当合作内容勾选「游戏代理金」时参与比对；未勾选时视为「-」 |
| 已付预付分成款 | 仅当合作内容勾选「预付分成款」时参与比对；未勾选时视为「-」 |
| 已付委托开发费 | 仅当合作内容勾选「委托开发费」时参与比对；未勾选时视为「-」 |

**不会**因以下字段单独变更而写入「合同变更」：合同编号、支付币种、合作内容勾选本身、补充说明。

若勾选/取消合作内容导致某项已付金额从有值变为「-」（或相反），按上表对应「已付…」字段记一条变更。

**示例（4012 农场物语）**：

```
"已付游戏代理金"变更为"￥40,000.00"
"已付委托开发费"变更为"￥85,000.00"
```

## 操作列展示规则

- **添加游戏**：固定文字。
- **运营状态 / 合作状态变更**：显示变更后的状态标签。
- **合同变更**：多行文字；金额带币种符号与千分位；未填显示「-」。

按操作时间从新到旧排序。无数据时显示「暂无操作记录」。只读，无保存操作。
